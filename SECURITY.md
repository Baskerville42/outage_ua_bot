# Політика безпеки

## Підтримувані версії
- Основна гілка `main`.
- Релізи (за наявності) — остання стабільна версія.

## Повідомлення про вразливості
Якщо ви виявили проблему безпеки:
- Не створюйте публічний Issue з деталями експлойту.
- Опишіть проблему з мінімальними деталями та позначте як «security», або
- Зв’яжіться приватно з власником репозиторію через профіль GitHub.

Будь ласка, надайте кроки відтворення, очікувану та фактичну поведінку, версію Node.js та середовище.

## Найкращі практики
- Не комітьте секрети (BOT_TOKEN) у код.
- Використовуйте GitHub Secrets у воркфлоу.
- Обмежуйте права бота в Telegram лише необхідними дозволами.
