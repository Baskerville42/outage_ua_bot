# Внесок у проект

Дякуємо за інтерес до GraphenkoBot! Нижче — короткі правила внеску.

## Як запропонувати зміни
- Спершу опишіть проблему або пропозицію в Issue.
- Для суттєвих змін — обговоріть ідею в Issue перед початком роботи.
- Форкніть репозиторій, створіть гілку з осмисленою назвою (`feature/...`, `fix/...`).
- Зробіть невеликий, логічно цілісний Pull Request з описом мотивації та змін.

## Налаштування середовища
- Потрібен Node.js 18+.
- Локальний запуск: створіть `.env` з `BOT_TOKEN=...`, виконайте `node index.mjs`.
- Автоматичні оновлення працюють через GitHub Actions (кожні 5 хвилин).

## Стиль коду
- JavaScript (ESM). Уникайте зайвих залежностей.
- Пишіть чіткі лог-повідомлення українською.
- Додавайте коментарі там, де є нетривіальна логіка.

## Тести та перевірки
- За можливості перевіряйте зміни локально (dry-run/логіка).
- Переконайтеся, що зміни не порушують роботу бота й формат `graphenko-chats.json`.

## Комунікація
- Дотримуйтеся [Кодексу поведінки](./CODE_OF_CONDUCT.md).
- Будьте ввічливі та конкретні в обґрунтуваннях.

## Ліцензія
- Надсилаючи зміни, ви погоджуєтеся з умовами ліцензії [MIT](./LICENSE).
